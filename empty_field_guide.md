# 空数据 / 留空字段 诊断对照表 (empty_field_guide.md)

> 核心原则：**先分清"空是对的"还是"空是错的"，再决定填不填。**
> 这个产品的灵魂是"诚实留空 > 编造填充"。很多空是**数据真相**，不是 bug——
> 把"诚实的空"当 bug 去填，就是在制造假数据，违背产品灵魂。
> 遇到字段空，先查下表对号入座，再动手。

## 诊断流程（按顺序）

1. **换一个已知有该数据的钱包对照**(见 test_wallets.md)：
   - 换钱包后字段就有了 → 是**这个钱包真没这数据**(诚实的空，别填)
   - 换钱包后字段还是空 → 才可能是**代码/口径 bug**(往下查)
2. 查参数真名是否写错(见 CLAUDE.md 的 [Heisenberg v3] 表)：
   - 参数名错会**静默返全局流或空**，第七道守卫(返回钱包≠请求钱包)能抓
3. 查口径/时间窗(见下表常见原因)
4. 确认是 bug 才改；是数据真相就**如实留空 + 标注原因**

## 常见空字段 → 原因对照

| 空的字段 | 最可能的原因 | 是真相还是 bug |
|----------|--------------|----------------|
| WHAT 区(建仓/均价/笔数全空) | 该钱包在**这个盘没有建仓记录** | ✅ 数据真相，留空 |
| 浮盈亏 = null | 市场**已结算**，无当前 K 线(用结算值 0/1 替代) | ✅ 真相，已有处理逻辑 |
| 催化剂数组 = [] | 该时点**真的没有相关新闻**(不是搜索失败) | ✅ 真相，诚实留空 |
| 负向催化剂栏空 | 该方向**真没威胁证据** vs **没搜到**——honesty_caveat 已区分 | ✅ 看 caveat 标的是 true_clearance 还是 search_vacuum |
| 市场反应 = 不可知 | as-of 防泄漏：新闻太靠近 as_of，"之后价格"窗口超过 as_of | ✅ 真相，诚实降级 |
| price_at 返回 None | 短命市场 T-7 未创建 / 新闻早于市场创建日 | ✅ 真相 |
| 569 某盘盈亏看着像 0 | **宽时间窗只返回前若干天**，结算日损失落在窗外 | 🔴 口径问题：�narrow 锚到结算期 |
| 581 返回空 | window_days 只支持 1/3/7/15，传别的值静默返空 | 🔴 参数问题 |
| 整个钱包数据全空/串了别人的 | **参数真名写错**(如 556 用了 wallet_proxy 静默返全局流) | 🔴 bug：第七道守卫该抓 |
| 排名查不到 | 584 H-Score 无按地址查；要按地址定位排名走 579 | 🔴 用错 endpoint |

## 给 Claude Code 的指令模板（遇到空字段时）

> "X 字段空了。先按 empty_field_guide.md 诊断：
> ①换一个 test_wallets.md 里已知有该数据的钱包对照，看是这个钱包真没数据、还是普遍空；
> ②若普遍空，查参数真名/口径/时间窗；
> ③明确告诉我这是'数据真相该留空'还是'bug该修'，再动手。
> 不要默认把空当 bug 去填——诚实留空是这个产品的灵魂。"
