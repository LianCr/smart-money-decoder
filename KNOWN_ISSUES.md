## 第一类 · 数据真实性与方法论（最核心，决定产品可信度）

### 1. 🔴 已结算"输盘"在公开接口几乎不留痕迹
- **问题**：份额归零、无事件、赎回后从 positions 消失。无法系统性纳入"聪明钱赌输"的样本。
- **后果**：回测天然偏向赢盘，命中率虚高。Starmer 两个亏损样本是碰巧抓到的，不可稳定复现。
- **修复方向**：调研第三方结算历史数据源（Dune / subgraph / 链上事件日志），或自建已结算市场结果库。
- **🔓 v3 解法已现（第九类）**：`556 Trades` 历史全成交 + `569 PnL` 真实盈亏，可重建含已赎回赢家的**无偏样本**，
  直击总根源。性质从"调研有没有数据源"→"验证这个 API 准不准、贵不贵"。
- **备注**：这是所有回测可信度问题的总根源，解决它能连带改善样本量、亏损样本、命中率口径。

### 2. 🟡 回测样本量过小且来源集中（6 个，实际仅 2 个钱包贡献）
- **问题**：4 个 whale 颗粒无收（政治大仓多为 2028 未结算盘 / 被短引信筛掉），有效样本仅伊朗 + Netanyahu。
- **后果**：统计上说明不了任何东西，命中率只能"看个意思"。
- **修复方向**：扩大钱包池 + 放宽筛选（见 #3），目标 ≥30–50 样本。

### 3. 🟡 "建仓时点历史价"要求筛掉了大量短引信盘
- **问题**：短引信盘恰是亏损样本聚集地，硬性时点门槛导致"难样本"系统性流失。
- **修复方向**：重新设计样本筛选，不以"能否拿到固定时点历史价"为硬门槛。

### 4. 🔴 decoder 在历史盘上几乎全给 NO BASIS（偏保守）
- **问题**：现有样本 decoder 大多判"别跟"。
- **后果**：好处是躲过 Starmer 的坑；坏处是可能"靠永远保守蒙对"，且错过 Powell 这类有效押注。当前是"风险过滤器"，不是"机会发现器"。
- **修复方向**：调研 NO BASIS 是因"真的没催化剂"还是"新闻检索能力不足"。若后者，加强新闻源（多语种 / 更多政治媒体 / 社交信号）可能解锁"敢跟"的判断。
- **🔓 v3 解法已现**：诊断已结案（见 v1 末期：4/6 是"那一刻真没新闻"、0/6 是检索不足）——"检索不足"那一支由
  **Polymarket Market Context（愿景 D，免费）/ 第九类 `585 Social Pulse`（付费）** 补强；"真没新闻"那一支无解
  （新闻当时尚未发生），但更强信息源能压缩误判 NO BASIS 的比例。**红线不变：绝不为敢跟而调松门槛。**
- **备注**：决定产品有没有"敢跟"的实用价值。若永远只会说别跟，产品价值存疑。

### 5. 🟡 难度系数暴露样本普遍"近明牌"
- **问题**：现有样本聪明钱多在 0.9+ 高价位建仓，难度系数普遍低（说明顶级玩家不爱赌五五开）。
- **后果**：回测页诚实显示"样本大多是简单题"。
- **修复方向**：主动寻找聪明钱在 0.4–0.7 区间建仓的样本——那才是真正能证明 decoder 判断力的考场。

## 第二类 · 回测引擎的简化与精度妥协

### 6. 🟡 回测重放时 resolution_criteria 传了 None
- **问题**：历史卡的 decoder 看不到结算细则，what_bet 质量比实时卡低。
- **修复方向**：回测重放时也带真实 resolution_criteria（数据已有，只是没传）。

### 7. 🔴 回测新闻锚的是建仓时间，不是各时点各自重搜
- **问题**：为省一半 API 调用，T-7 卡里的"新闻"不完全是 T-7 那一刻能看到的，有未来信息轻微泄漏风险。
- **修复方向**：T-7 / T-1 各自按其时点重新搜新闻并硬截断，精度优先于省调用。

### 8. 🔴 hit/miss 口径需锁死在早时点（T-7）
- **问题**：命中率必须算"价格未明牌的早时点"判断，不能算 T-1 送分时点。
- **状态**：v1 末期重构正在处理，需验证是否改干净。
- **修复方向**：固化口径，命中率永远基于早时点。**口径不硬，数字就是沙上城堡。**

### 9. 🟢 DURATION 守卫在批量重放时偶发误杀样本
- **问题**：日期型市场重试 3 次仍被守卫拦截，损失样本。守卫（防幻觉）与回测（需日期表述）的张力未完全解开。
- **修复方向**：给回测重放一个守卫软化 / 白名单模式，或让模型用受控格式表达日期冲突而不触发守卫。

## 第三类 · 正向流程（实时解读）的已知限制

### 10. 🟢 entry_time 命中率受限于数据特性
- **问题**：trades v2 解决大部分，但持仓极老 / 经 split-merge 获得的仓位仍拿不到建仓时间，只能降级。
- **状态**：已记为可接受降级。
- **修复方向**：v2 探索按市场维度的更深历史查询。

### 11. 🟢 单钱包 → 单卡，只看最大政治仓
- **问题**：一个钱包可能同时持有多个有意思的政治仓，现在只解读最大那个，信息有损。
- **修复方向**：支持展示 top N 政治仓，或让用户选。

### 11b. 🟡 "选盘规模最大" ≠ "最值得分析的盘"（选盘口径的根本局限）
- **已修的浅层 bug（2026-06-25）**：选盘曾按"近 60 天累计买入成本"排，**只算 BUY 不减 SELL** → 一个边买边卖、已基本清仓的盘会因历史买入额大被选成"最大仓"，给用户看个幽灵盘。已改为按**当前净持仓**(买入份额−卖出份额 × 均买价)排，幽灵盘自然落榜（实测 Abelardo 从已清仓的 "Democratic House" 修正为净持有的 "Netanyahu out"）。
- **更深的问题（没修、也不该在这里修）**：即便"净持仓最大"也不等于"最值得分析"——可能选中**已基本定局的稳赚盘**(价格贴近 0/1、无判断含量)，**漏掉有判断含量的小盘**(edge 空间大、早期进场)。
- **真正的解 = 「扫榜推荐」✅ 已落地（2026-06-26，方法 E 市场反向找大户）**：不在单钱包里改选盘算法，而是**从热门政治盘反向找共持大户**→质量门→⑥验证→推荐流主页。良率 12/30 vs 榜取顶 1/25。接红线 #6（最大仓≠最值得看的仓）。`recommend.py`·`fetcher/markets.py`·`hot_traders.py`。
- **现状兜底**：`R2`/`position_type` 已对对冲/做市盘降级提醒，不至于把对冲腿当信念注。

### 12. 🟢 resolution_criteria 缺失时 what_bet 降级
- **问题**：部分市场 Gamma API 不返回结算细则，规则套利分析无法进行。
- **修复方向**：多源补全结算规则，缺失时明确标注而非静默降级（目前已部分处理）。

### 13. 🟢 全链路十几秒，无法实时回测
- **问题**：正向解读尚可接受；回测换钱包要几分钟，塞不进 HTTP，所以回测页是预跑好的固定样本。
- **修复方向**：后台任务队列 + 异步通知，或预计算热门钱包。

## 第四类 · AI 输出的固有不确定性（只能管控，无法根除）

### 14. 🟡 模型对契约的服从不 100% 稳定
- **现状**：已部署六道代码守卫兜底——
  `INVALID_FOLLOW_CALL` / `CONFIDENCE_TAMPERED` / `FABRICATED_CATALYST` / `ENTRY_PRICE_DENIED` / `IRRELEVANT_CATALYST` / `DURATION_COMPUTED`。
- **问题**：守卫是"事后拦截"，模型仍会反复尝试违规（填满 catalyst、算 duration、混淆 entry_price/time）。
- **修复方向**：把守卫触发率本身作为监控指标，触发率高的规则说明 prompt 还能优化。
- **🔬 实例 + 数据卫生（2026-06-20，路 B 取数时发现）**：Powell 盘是非确定性的活标本——`.cache/backtest/result.json`（一次 run）判 **T-1=CHASED 跟了赢了**，而 git 跟踪正本 `cases.json`/`final_samples.md`（另一次 run）判 **T-1=NO BASIS 没跟（失手）**。两份卡各自内部自洽，根因是**两次不同 run + 模型在『临时主席算不算出局』这个真实结算歧义处翻了面**。
  - **数据卫生纪律**：`result.json` 是 .cache 残留、样本集都与正本不同（它有 Iran-Jun15、无 Project Freedom），**已与正本分叉、不可信，禁止在它上面建任何东西**。所有下游（路 B 等）**只读 git 跟踪的 `cases.json`/`final_samples.md`**。
  - **demo 含义**：Powell"失手"叙事仍是一次真实输出的如实记录、用于讲保守性 OK，但**不可复现**（重跑可能翻成 CHASED 赢）。被问"每次都这样吗"老实答"否，歧义处会抖"。

### 15. 🟡 catalyst 相关性判断仍偏松
- **问题**："允许放但要标 UNANCHORED"后，模型倾向放满 2 条（靠 IRRELEVANT_CATALYST 守卫兜）。
- **修复方向**：让模型对每条候选输出显式 relevance 评分，过阈值才进数组。

### 16. 🟡 依赖课堂网关 + Tavily，核心逻辑无法单测
- **问题**：单测只覆盖纯逻辑（JSON 解析 / 置信度矩阵 / 时间解析），AI 输出靠人工抽查 + 守卫兜底。
- **修复方向**：建"黄金样本集"（固定输入 → 人工标注的理想输出），做回归式质量监控。

## 第六类 · 产品叙事层面的根本张力（最值得想清楚）

### 22. 🟡 "解读聪明钱" vs "测 decoder 准确度"的口径漂移
- **问题**：产品初衷是"解读聪明钱的下注"，但回测因数据现实，实际测的是"decoder 判断准不准"。两者有缝——聪明钱可能判断对但 decoder 没看懂，反之亦然。
- **现状**：用"decoder 准确度"是诚实退路。
- **🔓 v3 解法**：路 B「离场盈亏」口径是这道漂移的终极解——回测从"测判断方向"升级到"跟着 AI 挑能赚多少钱"
  （第九类方向 #1 + `569 PnL realized`），叙事自然统一到"跟单收益"，不再二选一。

### 23. 🟡 难度系数公式是单一维度（只看价格距 0.5）
- **问题**：真实"判断难度"还包括新闻是否充分、规则是否清晰、事件是否突发。单维度会失真（一个 0.5 但有明确催化剂的盘其实不难）。
- **修复方向**：多维难度模型（价格不确定性 + 信息可得性 + 规则清晰度）。

## 第七类 · 顶层认知与世界观（从"赢家消失"发现中提炼）

> 来源：v2 回测取样体检发现 /holders 快照 30:70 偏向输家——赢家赎回离场后从链上消失，留下的是滞留持仓者。这不是一个孤立 bug，是这类产品的结构性宿命。以下是从中提炼的、应指导后续所有设计的顶层认知。

### 24. 🧭 数据视野结构性残缺（世界观，非待办功能）
- **认知**：产品只能观测"还没离场的滞留持仓者"，赢家干净利落赢了就走、记录被链上擦除。任何"读取当前链上状态"的工具，看到的都是幸存/滞留快照，不是完整历史真相。这与 #1（输盘读不到）是同一只幽灵的两次现身。
- **对设计的含义**：所有"胜率"类指标天生有幸存者偏差。产品定位必须建立在"诚实处理盲区"而非"还原真相"之上。界面应明确标注：看见了什么、漏掉了什么、在看不清时不假装看得清。
- **状态**：这是世界观，不是功能。已固化为产品定位句（见下方）。无需"实现"，需贯穿到文案、定位、demo 话术。

### 25. 🟡 "赎回/离场"是被忽略的最强信号（真功能，v2/v3）
- **洞察**：聪明钱赎回离场，等于用脚投票说"这注 edge 已尽、我走了"——这恰恰是产品核心问题"现在跟还来不来得及"的标准答案。但当前产品只看"他还持有什么"，完全没读"他正在离场吗"。
- **功能方向**：把聪明钱的减仓 / 赎回 / 清仓动作，作为**反向跟单信号**（他在离场 = 强烈的"别跟/该走"信号，比"他还拿着"信息量大得多）。
- **工程量**：中等——需要新的数据链（追踪持仓变化 / 赎回事件），是个独立功能，不是小改。
- **⚠️ 时机纪律**：v2 lift 已收尾立住，这条已**升级为 v3 路 B（离场盈亏）的核心信号**——见第八类愿景 A/B + 第九类
  `569 PnL realized`/`556 Trades`（离场动作 = 可捕捉的交易事件，数据现成）。纪律改为：**demo 后启动**。

### 26. 🧭 "清晰度"与"edge"互斥（预测市场根本悖论，影响定位）
- **认知**：等信息明朗到能"看清"一注时，市场已把价格推到终点、edge 也榨干了；有 edge 的早时点，又看不清楚。这正是 decoder "早时点没新闻看不清 / 建仓时点有新闻但会被骗"权衡的根源。
- **对设计的含义**：产品永远活在"清晰度 vs edge"的夹缝里，不存在"既确定又有空间"的免费午餐。诚实定位不是"帮你找稳赚机会"（那是骗人），而是"帮你理解这注的风险/空间结构"——卖的是清醒，不是确定性。
- **时间机制见 #27**：这个悖论的具体载体是"时间"——edge 浓在开盘头几小时、可解读性来得晚。

### 27. 🧭 alpha 主要来自"前 2–6 小时早进场"（时间错位，定位级认知，2026-06-15）
- **来源**：社区验证 50 个聪明钱钱包的共性——超额收益主要来自**新市场开盘后头 2–6 小时早进场**，不是"看懂了某一注"。
- **(a) 印证 v2 发现**：与 v2"只有建仓时点才有信号、解读时点信号已稀"完全同源——**edge 是会随时间蒸发的量，建仓那一刻最浓**。
- **(b) 暴露产品根本时间错位**：用户看到一个"可解读的持仓"时，往往已是开盘后很久，**早期 edge 常已被吃掉**——我们解读的是"信号的残骸"，不是"信号本身"。这是 #26（清晰度 vs edge 互斥）的**时间版具体机制**：要可解读就得等信息明朗，可一明朗 edge 就没了。
- **(c) 校准定位（诚实）**：现产品本质是**"风险过滤器"**（判断一个已存在的仓位还值不值得跟），**不是"早期机会发现器"**。后者需要**实时新开市场监控 + 秒级进场警报**，是**完全不同的产品形态**（哨兵，非解读器）。承接 #4 备注 + 下方定位句。
- **(d) 强化 v3 愿景 A（看动作）**：因此"**监控聪明钱在新市场的早期进场动作**"可能比"解读已有持仓"更接近**真 alpha**——这是范式从"看持仓快照"转向"看动作流"的最强论据。接第八类愿景 A + #25。
- **⚠️ 对设计的含义**：别假装现产品能抓早期 alpha；老实做好"风险过滤"这件能做的事，把"早期机会发现"明确划归 v3 哨兵形态。**这不是缺陷，是诚实的形态边界。**

### 28. 🧭 单仓视角会把"对冲/做市玩家"误判成"单边赌徒"（产品级认知，2026-06-20 Heisenberg 验证实锤）
- **来源**：v3 数据地基验证（`verify_heisenberg.py`）跑 ImJustKen 全量交易时撞见——v2 把他读成"持 **Yes** 单边赌 Starmer 下台、赌输的单边客"。**真相**：569/556 全量数据显示他在 Starmer 多个子盘 **Yes/No 两边都大额压**（754 笔 Starmer 交易里 Yes 502 / No 252；may-19 盘同时 Yes@0.024 size 10609 **和** No@0.92 size 11797）——他是**做市/对冲玩家**，不是单边赌徒。
- **误判根源**：公开 positions 只暴露"最大单仓"（Yes 那侧），把一个两边对冲的复杂玩家，**结构性地**压扁成"赌输的单边客"。这与 #1（输盘读不到）、#24（数据视野残缺）是同一只幽灵——**单点快照天生会误读玩家类型**。
- **对 v3 的含义（价值铁证）**：v3"看动作"范式（愿景 A）**必须用全量交易 + 净 PnL 识别玩家类型**，不能再用单仓视角解读画像。这条本身就是 **v3 相对 v2 的价值证明**——不是新数据更全那么简单，是**单仓视角会系统性骗你**，全量动作流才看得清"他到底是谁"。
- **⚠️ 诚实标注（demo 必须修正）**：现 demo 的 Starmer 案例叙事（样本 4/5，见 `backtest/final_samples.md`）**建立在这个误读之上**——"decoder 躲过单边赌下台的亏损"这个故事，事实层面是对的（那一侧确实亏），但**玩家画像是错的**（他不是单边赌徒）。v3 重做要修正叙事；v2 demo 若续用需口头标注此 caveat。接第八类愿景 A + #25。

### 愿景 A · 从"看持仓"转向"看动作"（范式转向）
- **触发认知**：96% 赢家离场，证明最有价值的信息不在"持有"里，在"进出"里。v1/v2 是"快照式"（抓此刻持有什么），但聪明钱的四个动作各自都是强信号，目前一个都没读：
  - 建仓 = 开始相信 / 加仓 = 信念增强 / 赎回 = edge 已尽该走 / 割肉离场 = 认输看错
- **转向**：v3 范式从"持仓快照"→"行为流"。不问"他现在拿着什么"，问"他刚做了什么动作、意味着什么"。
- **反转盲区为信号源**：赢家"消失"让你看不到持仓，但"离场"这个动作本身是可捕捉的交易事件——把最大的盲区（赢家消失）变成最强的信号（"该走了"）。接 #25。
- **🔺 最强论据（2026-06-15，#27）**：社区验证 alpha 主要来自新市场开盘头 2–6h 早进场——"监控聪明钱在新市场的**早期进场动作**"比"解读已有持仓"更接近真 alpha。**看动作是唯一能在 edge 蒸发前抓到它的形态**；解读已有持仓注定慢半拍（#27b 时间错位）。
- **✅ 已落地（2026-06-22）：巨鲸 48h 行为流**（`556 Trades`，`briefing/market_context.get_behavior_flags`）：分 3h/24h/48h 窗 → `ADD`信念增强 / `EXIT`主力撤退(该走) / `STATIC`诚实留白(真空+加仓→内幕/薄盘提示)。四个动作里"加仓/退出"已读到、并作**最高权重信号**喂进 Context 综述 + 统一看板 ④。**纪律**：渲染成**事实**("过去 24h 大额 SELL 流…由你裁决")，不替用户判断。**未做**：跨市场"新市场早期进场"哨兵(愿景 B 的实时订阅形态)。

### 愿景 B · 从"解读单点"升级到"追踪轨迹"（产品形态升级）
- **触发认知**：聪明钱的 alpha 不在某一注，在完整决策轨迹（何时进、何价加、看到什么走）。
- **升级**：解决的问题从"这一注在赌什么、还能不能跟"→"追踪一个聪明钱的完整下注生命周期，在每个关键动作点提醒你"。从"我帮你读懂这一注"→"我帮你像影子一样跟住这个人"。
- **形态变化**：v3 可能不再是"输入钱包出张卡"，而是"订阅一个聪明钱，他一动你就知道"——从工具变哨兵。接 #13（无法实时）+ #25。
- **🔺 落地口径（第十类 反馈 1）**：导师把"离场盈亏"翻成用户能懂的话——"AI 判 ROOM LEFT/CHASED 时跟入 $1000 的平均收益率 vs 无脑全抄的平均收益"。这是路 B 的**可执行验收口径**，**v3 第一仗**。
- **✅ 数据地基已验证可开工（2026-06-20）**：路 B 命门"离场/已实现盈亏拿不拿得到"已用 Heisenberg `569 PnL` 实测通过（详见第九类验证战报）——**569 完整含『持到归零』全损**，跨 2 个结算日 23/24 个干净单边输方精确到分。**路 B 绿灯。**
  - **🔧 实现纪律（验证副产物，钉死）**：路 B 算收益率时，对**被跟的具体仓位**用 `556 Trades + 574 Markets 结算结果` **自重建已实现盈亏**（确定性：`payout − cost`，完整），**569 只做交叉校验**。原因：569 的 per-condition_id 归因对"单日结算上千仓的超高频 bot"会漏掉个别尘埃仓（实测 1 个 $0.10 仓，钱包级仍可见）。自重建把这 4% 归因风险归零；聪明钱目标都是有意仓位，实际几乎不影响，但纪律照立。

### 愿景 C · 终极问题不是"跟谁"，是"信谁"（核心引擎升级）
- **触发认知**：v2 全程真正在打磨的核心能力，不是"解读下注"，是"判断可信度"（红线、防泄漏、去偏差、辨真假催化剂）。预测市场里真正稀缺的不是信息，是"哪些信息/哪些人值得信"。
- **升级为"可信度引擎"**：
  - 不只"聪明钱买了什么"，而是"这个聪明钱在**这类盘**上历史上靠不靠谱"（有人擅长地缘、有人擅长选举，不该一视同仁）
  - 不只给新闻，而是"这条新闻是真催化剂还是会骗你的噪音"（decoder 已有雏形，如 Starmer 那条骗人的辞职压力新闻）
  - 不只算 lift，而是"什么条件下我的判断可信、什么条件下我自己也没把握"（置信度的下一代）
- **本质**：卖的从来不是"信号"，是"信号的可信度"。是定位句"卖清醒不卖确定"的终极工程化形态。
- **✅ 雏形落地（2026-06-23）：⑥ Edge/Reasoning = 可信度引擎第一步**（`analyzer/reasoner_v3.py`）：
  - "这类盘靠不靠谱"→ 统一看板 ① 已接官方榜 + **政治盘专长**(category_specialization)，胜率谎言守卫(高胜率但净亏→看净盈亏)。
  - "真催化剂还是骗你的噪音"→ 双向催化剂 `material_tag` + **市场测谎仪**(market_reaction:印证/反定价/微弱)，并作 `R1` 进矩阵。
  - "什么条件下我可信"→ **新 v3 置信度矩阵**：v2 底座删 rule5 + `R1`测谎背离/`R2`对冲/`R3`退出/`R4`双空 只降不升 + **降级原因列表**(把"我为什么没把握"显式说出)；reasoner 守三铁律(不判对错/测谎≠水平/不替用户决定)。
  - **未做**：跨"盘类"的可信度分层(现仅政治盘专长一维)、Falcon Score(`584`)/Wallet 360(`581`) 接入做可信度增强。
- **🔴 重设计落地（2026-06-26）：信心从"钱包方向归因"换成"市场命题级裁决"**（`analyzer/market_thesis.py`，dashboard ⑥ 信心已切到它，老 R1-R4 矩阵下沉为 fallback）：
  - **病根**：旧矩阵锚钱包 pnl → 同一盘两个反向钱包"证据反对却因 +10% 浮盈给高信心"（`_market_thesis_probe` 实证）。**信心的病一直是"信错了不该全信的信号"——pnl 是一例。**
  - **新法**：同一文章池 bull(YES)‖bear(NO)→reasoner 中立裁决**直出单一信心**；按 `(cid,as_of)` 缓存→两反向钱包共享同一份市场观、信心一致，差异挪到"顺/逆 edge"。撤代码兜底守卫(产品决策)，改为**可观测**(记 `confidence_log.jsonl` 待结算回验)。
  - **可信度修正（这才是上面"未做"的真正解，且比 584/581 更直接）**：核心原则**"信心被最可信的锚封顶、两弱信号一致≠可信"**——575 价格可信度(薄盘/鲸控→价格降权)+568 已实现波动(市场犹豫度)+距结算时间；多结局事件(574 `event_slug`)按 1/N 基线读不当二元。Kalshi 跨场复现验证后砍(无政治覆盖)。
  - **专家共识(581 技能)留在推荐层、不进信心**——与鲸控价循环(同批鲸鱼既推价又算"专家")，只当弱信号 + 与同盘分歧对称。
- **✅ 自我验证落地（2026-06-23）：诚实记分牌**（`scorecard.py`）：可信度引擎的闭环——不只"判断时给信心"，更"判断**后**回来对账,看我的判断方向被现实证明对了多少"。decode/board 判断存档(装上后累积、绝不回填)→ 574 抓结算 → 纯代码命中率(**非收益率**)、NO BASIS 单列自审。**待验证**:真实时间推进下的自动填充闭环尚未观察(开放盘暂全待结算)。这是定位句"卖清醒不卖确定"第一次能**被自己量化检验**。

### 愿景 D · 换掉弱新闻源：Polymarket 盘面 Market Context（decoder 输入升级，2026-06-15）
- **发现**：Polymarket 盘面自带 AI 生成的新闻摘要 + 时间线 + 价格跳变标注，比现用 Tavily 通用搜索**精准对题得多**。
  实测到一个钱包 Tavily 搜不到催化剂，但同一盘 Polymarket 盘面新闻很丰富。
- **关键认知（别误读成竞品）**：这是**免费的优质原料**。Polymarket 做"陈列新闻"（平台不能给投资判断），
  我们做"读完新闻 + 看聪明钱行为 + 判断该不该跟"——它新闻越丰富，越替我们干了"找新闻"的脏活，我们专注它做不了的判断层。
- **与第九类的关系（两条候选解法，别当重复）**：都为换掉 Tavily——
  **Market Context（盘面自带、免费）** vs 第九类 **`585 Social Pulse`（X 情绪/动量/叙事vs价格背离、付费 API）**；
  二者互补，可二选一或并用，都强于 Tavily 通用搜索。直接缓解 #4 的"检索不足"那一支。
- **✅ 前置验证已做（2026-06-20，免费 curl gamma API，结论=部分存在）**：
  - **存在且是干净免费公开 API**：`gamma-api.polymarket.com/events` 的 **`eventMetadata.context_description`**（无需认证、稳定 JSON 字段、**无需 JS 渲染/脆弱抓取**）。心结了结：它真的有。
  - **但比当初想的朴素**：eventMetadata 只有 3 个字段 `context_description`(一段 AI 叙事 prose) + `context_requires_regen` + `context_updated_at`。**没有**离散的新闻链接、**没有**推特链接、**没有**时间线结构——就是**一段散文摘要**（含赔率/近期发展/比分等，对题性强，因为是这个 event 自己的叙事）。
  - **两个边界**：① **event 级**（父级一段，覆盖其所有子市场），非 per-market；② **当前快照**（`context_updated_at` 周期重生成）——**同 585，只能实时、不能回测 as-of**（拿不到"某历史时点的 context_description"）。
  - **裁决**：达到"干净/稳定/免费/公开 API"门槛 → **可纳入**，但定位为**实时简报的一个免费补充叙事源**（喂双向催化剂框架当额外原料），**不进回测**；它是锦上添花不是雪中送炭（新闻主力仍 Tavily）。**不做任何前端 JS 抓取。**
  - **集成利好**：gamma 返回的 `context_updated_at`=2026-06-18、event=World Cup 2026 → gamma 与 Heisenberg 同处 2026 数据世界，集成无时间线撕裂。
- **⚠️ 时机**：动 decoder 输入管道 = 全链路回归风险，**demo 前绝不碰**，v3 启动时再做。
- **✅ 落地走了另一条路（2026-06-20→23）：GDELT 三层管线 + Context「一虚一实」**——没用 Tavily/也没只靠 `context_description`，而是用**免费 GDELT GKG**(海量召回→实体硬过滤→LLM 重排，as-of 物理防泄漏)自建对题催化剂源；前端 Context 视图把 **Polymarket 实时盘面 iframe(实)** 与 **as-of 复盘(虚)** 同框。**钉死的坑**：GDELT 实体聚合(tone/volume)粒度太粗、**不能当市场交叉判定**(Burnham +35% 跳变 tone/volume 双哑火)→ GDELT 只当催化剂来源，测谎维持单层(盘价 × 单条新闻)。`context_description` 作为额外叙事源仍可选纳入(未集成)。

### 它带来的一个更深的战略问题（值得想清楚）
- 如果这个 API 把输盘/PnL/可信度评分/情绪全给了我们，**别人也能买到同样数据，护城河在哪？**

## 第十类 · 导师评审反馈（外部资深视角，2026-06-15）

> 来源：外部资深导师评审，**分量重**。原则照旧——**不重复造轮子**：这 7 条几乎每一条都对接到既有的
> 第七类世界观 / 第八类 v3 愿景 / 第九类数据 API，把它们焊到现有结构上，而不是另起孤岛。
> 一句话定调：导师没有推翻方向，而是**把"路 B / 看动作 / 可信度"用更落地、更用户视角的语言重说了一遍**。

### 反馈 1 · 🔴【最重要 · performance 量化升级】从"判断准确率"到"跟单收益率"
- **导师指出**：现 backtest 测"某时点判断方向对不对 vs 真实结果"——**没说服力**，因为 LLM 开 deep search 也能做方向判断，无额外价值。
- **导师解法（比原「路 B」更落地）**：当 AI 判 `ROOM LEFT`/`CHASED`（有信心跟）时，假设**跟入 $1000、算这笔最终收益**；所有这类案例求**平均收益率**。再加**对照组**：同样 $1000 无脑全抄聪明钱每注的平均收益。两者一比 = **AI 筛选的真实价值**。
- **为何吊打现版**：从"判断准确率"（学术指标）→"跟单收益率"（用户唯一关心：跟你能赚多少钱）。这是 **#22 口径漂移的终极解**，把 lift 从"方向胜率"升级成"真金白银收益率"。
- **数据前提**：需聪明钱真实进出盈亏 = 公开 API 拿不到（#1 / 第七类 #24）= 正是**第九类 `569 PnL` + `556 Trades`** 提供的。
- **定位**：这就是**第八类愿景 B / 路 B（离场盈亏）的落地形态**，导师帮我们说得更具体可执行。**v3 第一仗**（具体化「v2 三大主攻」第 3 条的 round-trip 口径）。

### 反馈 2 · 🔴【灵魂 · 产品价值定位】deep search 也能搜新闻，凭什么是你？
- **导师一针见血**：LLM deep search 就能搜新闻 + 分析，你的 product 价值在哪？
- **我们的三层答案（写进文档当定位论据）**：
  1. deep search 能搜新闻，但**不知道"某个赚了百万的聪明钱真金白银押了这一注"**——行为信号 **who**，只有接链上 / 第九类数据才有。
  2. deep search **不知道"这注还剩多少空间、入场价 vs 现价"**——**how much**，是代码算的 `price_info`、防幻觉。
  3. deep search 给信息，但**不判断"该信几分"**——**可信度 = 我们的灵魂**（愿景 C + 定位句"卖清醒不卖确定"）。
- **核心结论**：我们卖的不是"搜新闻"，是 **"聪明钱行为(who) + 新闻(why) + 价格空间(how much) 三合一的可信判断"**；deep search 只有第二个 why。
- **行动含义**：必须靠"聪明钱真实行为"这层和 deep search 拉开差距——**反馈 3 / 4 都是在加厚这一层**。呼应第九类战略问题"护城河不在数据、在判断"。

### 反馈 3 · 🟢【功能】主页定时挑高信心钱包（从"你输我析"→"我主动帮你挑"）
- **做法**：定时任务从榜单拉钱包 → 跑分析 → 挑出 AI 判 `ROOM LEFT` 且高信心的 ~10 个 → 放主页。
- **价值**：产品形态从被动"你输我析"升级到主动"我帮你挑"，更像产品。
- **成本**：定时任务 + 缓存 + 烧 token（**额度紧张、需预算**；接 #19 离线刷新、#21 缓存/限流）。
- **定位**：v3 功能。

### 反馈 4 · 🟢→重头戏【功能】subscribe 钱包 + 卖出警告（哨兵形态）
- **做法**：订阅钱包 → 定时监控 → 负面新闻 / 价格大跌 → 推**卖出警告**。
- **直接对接**：第八类**愿景 B"从工具变哨兵"** + 第七类 **#25"赎回/离场是最强信号"**——subscribe + 离场检测是同一方向。
- **工程量**：大（后台监控 / 推送 / 状态机），是真功能不是小 feature；接 #13（无法实时）。
- **定位**：**v3 重头戏。**

### 反馈 5 · 🟡【判断质量】双 LLM 搜正/反 + 第三个整合
- **做法**：一个 LLM 专搜**支持**这注的新闻、一个专搜**反对**的、第三个整合判断。
- **价值**：强制 AI"两面都看"，judgment 更平衡、更难被单边假利好骗——**直接缓解 Starmer 那种"被假利好骗"风险**（愿景 C"辨真假催化剂" + #15 相关性偏松）。
- **成本**：LLM 调用 ×3、token ×3。判断质量↑但烧钱↑。
- **定位**：v2.5 / v3。
- **🛠 MVP 已动工（2026-06-20，`analyzer/dual_catalyst.py`）**：做成 **2-LLM 版**（砍掉"第三个整合 LLM"——整合=拍板=违反红线 #3；改 2 个独立 LLM 出证据 + 代码机械天平 + 用户裁决，比原版更贴产品魂）。双向独立搜索（正向/威胁向各一次 Tavily，否则负向栏永远空）。三道代码守卫：恐吓词表（负向 LLM 是"冷静尽调员"非"检察官"，禁致命/扼杀/毁灭等情绪化修辞）/ 导向词（禁建议跟单/胜率）/ honesty_caveat 代码判定（true_clearance vs search_vacuum）。
- **🔭 范围纪律 + 下一步（框架验证后才做，不是现在）**：MVP **只用 Tavily 一个 source**，先验证"AI 在双向辩证上能不能做得既深又诚实"这个**框架**。框架若 work → 再把 **585 Social Pulse（实时社交动量，补 Tavily 对早期推特信号的滞后）** + **愿景 D 的 Polymarket Market Context** 作为额外 source 接进这个**已验证的框架**。若 AI 拿 Tavily 都做不好双向辩证 → 加再多 source 也白搭。**先验框架、再堆 source。**
- **✅ MVP 成立（2026-06-20，3 版迭代后收尾）**：Starmer 案例实测，负向 LLM 独立挖出"他拒绝辞职"那个决定性反面（正是当年判 NO BASIS、赌注真输的核心信号），且把"辞职压力(看着利好)"与"他拒绝(空头真相)"显式拆开摆给用户——**做到了 deep search 做不到的辩证组织**。守卫齐：相关性门（核心实体须在标题，挡边角混入）/ 恐吓词 / 导向词 / 类型须属固定集 / 份量词扫 type+reason / honesty_caveat 代码判定。单案例约 3200 老师 token。
- **💎 意外红利：透明本身就是纠错机制**：材质标签自己抓出了"负向方向错"的漏洞——把"推他下台的辞职压力"误收进威胁栏时，材质显得违和、一眼可疑。**摊开摆证据的系统，比给结论的系统更容易自我纠错**。这是"卖清醒不卖确定"路线的结构性优势，不只是合规姿态。
- **🛑 收尾纪律**：负向"方向正确性"用**够用版**（prompt 方向铁律 + 材质标签兜底漏网的），**不追求 100% 纯净**——方向是语义判断、AI 做不到 100% 准（同 decoder 歧义处非确定性 #14），追求完美纯净是无底洞。**MVP 已成立、停止打磨**；下一步是"接进完整产品 / v3 整体方向"，不是继续优化本模块。

### 反馈 6 · 🟢【机制】回测偏差反哺 reasoning（持续优化）

- **前提**：需足够回测样本后才有意义（接 #2 样本量）。
- **定位**：v3 持续优化机制。

### 反馈 7 · 🔴【与反馈 2 一体】新闻源升级

### 同时修正 · 信心分设计（避免"假精确"埋雷）
- **现状**：信心分 = 代码按"这注盈亏状态 + 有无新闻证据"算（置信度矩阵）。
- **v3 增强方向**：想纳入"这个钱包在相关话题的历史胜率"作为信心因子——**方向对**。
- **⚠️ 但必须诚实处理**：**不能用公开数据硬算这个胜率**。赢家赎回离场后链上消失（**第七类 #24 数据视野残缺 / 96% 赢家消失**），公开 `positions` 算出的胜率被**幸存者偏差**污染、是**假精确**，塞进信心分会**违背产品诚实**（定位句的反面）。
- **正确做法**：用第九类经处理的可信第三方评分增强信心——**`584 Falcon Score`（交易员质量分）/ `581 Wallet 360`**，而非自己用公开 positions 硬算胜率。
- **定位**：v3 用可信第三方质量评分增强信心分；**在此之前不纳入"自算胜率"，避免假精确**。这是 #24 世界观在信心分设计上的具体落地纪律。
