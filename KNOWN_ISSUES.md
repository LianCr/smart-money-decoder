## v2 结算结论（2026-06-14 · 钉死，勿再翻案）

> 完整数据 + 三层裁决见 `backtest/lift_v1.md`。**v2 到此收尾**——这个诚实的三层裁决就是 v2 的成果，
> 不追求更漂亮的数字。路 A 取样管道首次跑出 lift（市场优先 / N=94 / 27 盘 / 建仓时点重放 /
> 结算输赢口径）：全集 GO 94% vs 基线 84% = **+10%**；edge-band(0.10–0.90, N=30) GO 67% vs 53% = **+13%**。

**三条钉死结论：**

1. **decoder「诚实保守、不瞎跟」= 已验证成立。** 量化：94 盘只 GO 17，真五五开的 edge-band 30 盘
   只 GO **3**。它不是橡皮图章、高度选择性，红线保守性被数据证实。**这一半（产品的「清醒」内核）立住了。**

2. **decoder「能否在硬盘发现可盈利 edge」= 测不出，但不是证伪。** 被「结算输赢口径在 edge-band 只有
   3 个 GO」饿死：2/3 翻成 1/3 就反号、统计等于没说；它躲掉的硬盘赢 52%≈基线 53%、聚合不带方向信息。
   **不是 decoder 没 edge，是这个口径喂不饱数据**（近明牌占满 68% 样本抬高基线、真考场 edge-band 的 GO 太稀）。

3. **救它的明确路径 = 换「离场盈亏」口径（路 B），而路 B = v3 愿景 A/B（看动作 / 追踪轨迹）同一个工程。**
   离场盈亏让每笔 round-trip 都是合法样本（不需持到结算）、幸存者偏差从根上消失、硬盘样本不再被
   「持到结算」过滤 → GO 子集喂得饱。**v2 的数据亲手把路 B 从「可选」升级成「v3 首要任务」——有数据依据，不是直觉。**

**时机纪律**：先不动手做路 B；v2 收尾、结论固化；路 B 作为 **v3 第一仗**，等整体节奏想清楚再启动。
**红线（不变）**：绝不为让 lift 更好看去调松 decoder 跟单门槛——保守正是结论 1 成立的原因。

## 第一类 · 数据真实性与方法论（最核心，决定产品可信度）

### 1. 🔴 已结算"输盘"在公开接口几乎不留痕迹
- **问题**：份额归零、无事件、赎回后从 positions 消失。无法系统性纳入"聪明钱赌输"的样本。
- **后果**：回测天然偏向赢盘，命中率虚高。Starmer 两个亏损样本是碰巧抓到的，不可稳定复现。
- **修复方向**：调研第三方结算历史数据源（Dune / subgraph / 链上事件日志），或自建已结算市场结果库。
- **备注**：这是所有回测可信度问题的总根源，解决它能连带改善样本量、亏损样本、命中率口径。

### 2. 🟡 回测样本量过小且来源集中（6 个，实际仅 2 个钱包贡献）
- **问题**：4 个 whale 颗粒无收（政治大仓多为 2028 未结算盘 / 被短引信筛掉），有效样本仅伊朗 + Netanyahu。
- **后果**：统计上说明不了任何东西，命中率只能"看个意思"。
- **修复方向**：扩大钱包池 + 放宽筛选（见 #3），目标 ≥30–50 样本。

### 3. 🟡 "建仓时点历史价"要求筛掉了大量短引信盘
- **问题**：短引信盘恰是亏损样本聚集地，硬性时点门槛导致"难样本"系统性流失。
- **修复方向**：重新设计样本筛选，不以"能否拿到固定时点历史价"为硬门槛。

### 4. ✅ decoder 在历史盘上几乎全给 NO BASIS（2026-06-13 已诊断结案）
- **问题**：现有样本 decoder 大多判"别跟"。
- **✅ 诊断结论（在 #7 干净新闻上做 a/b）**：早时点 NO BASIS **4/6 是 (a) 那一刻真没新闻，0/6 是
  decoder 对支持性新闻过严**。冒烟的枪：同一批 Starmer 辞职新闻，对 May15 盘在 T-7 之后、对 May31 盘在
  T-7 之前。→ **decoder 没病、保守是对的（红线被证实正当，绝不调松）；「加强新闻源」是死路（a 是新闻
  尚未发生，检索不到不存在的新闻）；真正的杠杆是「时点定义」——T-7（结算-7天）对短引信盘太早、早于催化剂。**
- **→ v2 新方向**：早时点从「固定结算-7天」改为锚定**建仓时点 entry_time**（最贴"解读聪明钱"灵魂）。
  换时点 GO 率从「结算-7天」的 0/6 升到「建仓时点」的多数有 GO（`backtest/_scan_timepoint.py` 已验证）。
- **✅✅ v2 终局结论（2026-06-14，见顶部「v2 结算结论」）**：保守性在 N=94 上**验证成立**（只 GO 17、
  edge-band 30 只 GO 3）；但「decoder 能否在硬盘发现可盈利 edge」被**结算口径 GO 太稀（3）饿死、测不出**
  （非证伪）。下一步换「离场盈亏」口径（路 B）才喂得饱——红线不变，绝不为提 GO 率调松门槛。

### 5. 🟡 难度系数暴露样本普遍"近明牌"
- **问题**：现有样本聪明钱多在 0.9+ 高价位建仓，难度系数普遍低（说明顶级玩家不爱赌五五开）。
- **后果**：回测页诚实显示"样本大多是简单题"。
- **修复方向**：主动寻找聪明钱在 0.4–0.7 区间建仓的样本——那才是真正能证明 decoder 判断力的考场。

## 第二类 · 回测引擎的简化与精度妥协

### 6. 🟡 回测重放时 resolution_criteria 传了 None
- **问题**：历史卡的 decoder 看不到结算细则，what_bet 质量比实时卡低。
- **修复方向**：回测重放时也带真实 resolution_criteria（数据已有，只是没传）。

### 7. ✅ 回测新闻锚的是建仓时间，不是各时点各自重搜（2026-06-13 已落地）
- **问题**：为省一半 API 调用，T-7 卡里的"新闻"不完全是 T-7 那一刻能看到的，有未来信息轻微泄漏风险。
- **✅ 已修**：`fetcher/news.py` 的 `get_news_for_market`/`_build_time_window` 加可选 `as_of`（unix 秒），
  回测各时点各自重搜、end 截到该时点（`[建仓-7, as_of]`），现有 `end_date` 过滤即硬截断；窗退化（建仓晚于
  时点）时不走 30 天兜底（那会泄漏）、返回空新闻。正向 `as_of=None` 行为不变。重跑验证 0 泄漏。

### 8. 🔴 hit/miss 口径需锁死在早时点（T-7）
- **问题**：命中率必须算"价格未明牌的早时点"判断，不能算 T-1 送分时点。
- **状态**：v1 末期重构正在处理，需验证是否改干净。
- **修复方向**：固化口径，命中率永远基于早时点。**口径不硬，数字就是沙上城堡。**
- **✅ v2 进展（2026-06-14）**：lift 口径取代旧 hit——用**建仓时点**（最早、最干净）+ **GO 子集方向
  胜率 vs 全集**，不再算 T-1 送分 hit。但 v2 数据揭示更深一层：**「结算输赢」口径本身**在 edge-band
  GO 太稀（3）→ 喂不饱，下一代口径 = **离场盈亏**（见「v2 结算结论」/ 愿景 A）。

### 9. 🟢 DURATION 守卫在批量重放时偶发误杀样本
- **问题**：日期型市场重试 3 次仍被守卫拦截，损失样本。守卫（防幻觉）与回测（需日期表述）的张力未完全解开。
- **修复方向**：给回测重放一个守卫软化 / 白名单模式，或让模型用受控格式表达日期冲突而不触发守卫。

## 第三类 · 正向流程（实时解读）的已知限制

### 10. 🟢 entry_time 命中率受限于数据特性
- **问题**：trades v2 解决大部分，但持仓极老 / 经 split-merge 获得的仓位仍拿不到建仓时间，只能降级。
- **状态**：已记为可接受降级。
- **修复方向**：v2 探索按市场维度的更深历史查询。

### 11. 🟢 单钱包 → 单卡，只看最大政治仓
- **问题**：一个钱包可能同时持有多个有意思的政治仓，现在只解读最大那个，信息有损。
- **修复方向**：支持展示 top N 政治仓，或让用户选。

### 12. 🟢 resolution_criteria 缺失时 what_bet 降级
- **问题**：部分市场 Gamma API 不返回结算细则，规则套利分析无法进行。
- **修复方向**：多源补全结算规则，缺失时明确标注而非静默降级（目前已部分处理）。

### 13. 🟢 全链路十几秒，无法实时回测
- **问题**：正向解读尚可接受；回测换钱包要几分钟，塞不进 HTTP，所以回测页是预跑好的固定样本。
- **修复方向**：后台任务队列 + 异步通知，或预计算热门钱包。

## 第四类 · AI 输出的固有不确定性（只能管控，无法根除）

### 14. 🟡 模型对契约的服从不 100% 稳定
- **现状**：已部署六道代码守卫兜底——
  `INVALID_FOLLOW_CALL` / `CONFIDENCE_TAMPERED` / `FABRICATED_CATALYST` / `ENTRY_PRICE_DENIED` / `IRRELEVANT_CATALYST` / `DURATION_COMPUTED`。
- **问题**：守卫是"事后拦截"，模型仍会反复尝试违规（填满 catalyst、算 duration、混淆 entry_price/time）。
- **修复方向**：把守卫触发率本身作为监控指标，触发率高的规则说明 prompt 还能优化。

### 15. 🟡 catalyst 相关性判断仍偏松
- **问题**："允许放但要标 UNANCHORED"后，模型倾向放满 2 条（靠 IRRELEVANT_CATALYST 守卫兜）。
- **修复方向**：让模型对每条候选输出显式 relevance 评分，过阈值才进数组。

### 16. 🟡 依赖课堂网关 + Tavily，核心逻辑无法单测
- **问题**：单测只覆盖纯逻辑（JSON 解析 / 置信度矩阵 / 时间解析），AI 输出靠人工抽查 + 守卫兜底。
- **修复方向**：建"黄金样本集"（固定输入 → 人工标注的理想输出），做回归式质量监控。

## 第六类 · 产品叙事层面的根本张力（最值得想清楚）

### 22. ✅ "解读聪明钱" vs "测 decoder 准确度"的口径漂移（2026-06-13 已拍板）
- **问题**：产品初衷是"解读聪明钱的下注"，但回测因数据现实，实际测的是"decoder 判断准不准"。两者有缝——聪明钱可能判断对但 decoder 没看懂，反之亦然。
- **✅ 定位拍板 = C「聪明钱筛选器」**：叙事「我不号称比鲸鱼聪明，我告诉你它哪注还值得跟」。decoder 是
  聪明钱的二次过滤器，钱包出注 + decoder 筛注，两边都诚实归功——缝合此缝。
- **回测口径（C-降级，不依赖 #1）**：「GO 子集方向胜率 vs 全集方向胜率」的 lift（提升），只需公开结算结果。
  #1 输盘数据有解再升级到「真实回报 lift」（C-完整）；无解则永久停在 C-降级，不强求。

### 23. 🟡 难度系数公式是单一维度（只看价格距 0.5）
- **问题**：真实"判断难度"还包括新闻是否充分、规则是否清晰、事件是否突发。单维度会失真（一个 0.5 但有明确催化剂的盘其实不难）。
- **修复方向**：多维难度模型（价格不确定性 + 信息可得性 + 规则清晰度）。

## 第七类 · 顶层认知与世界观（从"赢家消失"发现中提炼）

> 来源：v2 回测取样体检发现 /holders 快照 30:70 偏向输家——赢家赎回离场后从链上消失，留下的是滞留持仓者。这不是一个孤立 bug，是这类产品的结构性宿命。以下是从中提炼的、应指导后续所有设计的顶层认知。

### 24. 🧭 数据视野结构性残缺（世界观，非待办功能）
- **认知**：产品只能观测"还没离场的滞留持仓者"，赢家干净利落赢了就走、记录被链上擦除。任何"读取当前链上状态"的工具，看到的都是幸存/滞留快照，不是完整历史真相。这与 #1（输盘读不到）是同一只幽灵的两次现身。
- **对设计的含义**：所有"胜率"类指标天生有幸存者偏差。产品定位必须建立在"诚实处理盲区"而非"还原真相"之上。界面应明确标注：看见了什么、漏掉了什么、在看不清时不假装看得清。
- **状态**：这是世界观，不是功能。已固化为产品定位句（见下方）。无需"实现"，需贯穿到文案、定位、demo 话术。

### 25. 🟡 "赎回/离场"是被忽略的最强信号（真功能，v2/v3）
- **洞察**：聪明钱赎回离场，等于用脚投票说"这注 edge 已尽、我走了"——这恰恰是产品核心问题"现在跟还来不来得及"的标准答案。但当前产品只看"他还持有什么"，完全没读"他正在离场吗"。
- **功能方向**：把聪明钱的减仓 / 赎回 / 清仓动作，作为**反向跟单信号**（他在离场 = 强烈的"别跟/该走"信号，比"他还拿着"信息量大得多）。
- **工程量**：中等——需要新的数据链（追踪持仓变化 / 赎回事件），是个独立功能，不是小改。
- **⚠️ 时机纪律**：**不在 v2 lift 主线收尾前碰它**。这是个诱人的新方向，在收口阶段开它 = 功能蔓延、断主线。记在这里等 lift 算完、v2 回测立住后再启动。

### 26. 🧭 "清晰度"与"edge"互斥（预测市场根本悖论，影响定位）
- **认知**：等信息明朗到能"看清"一注时，市场已把价格推到终点、edge 也榨干了；有 edge 的早时点，又看不清楚。这正是 decoder "早时点没新闻看不清 / 建仓时点有新闻但会被骗"权衡的根源。
- **对设计的含义**：产品永远活在"清晰度 vs edge"的夹缝里，不存在"既确定又有空间"的免费午餐。诚实定位不是"帮你找稳赚机会"（那是骗人），而是"帮你理解这注的风险/空间结构"——卖的是清醒，不是确定性。

## 产品定位句（从第七类认知提炼，用于 demo / 申请 / 对外）

> **一句话**：AlphaLens 不卖确定性，卖的是对不确定性的清醒。
>
> **完整版**：产品有一个诚实的世界观——它知道自己看不到完整真相（赢家赎回离场后链上记录被擦除，留下滞留者的残影）。所以它不假装能预测，而是帮你理解每一注的风险与空间结构，并明确标出自己看见了什么、漏掉了什么。价值不在"告诉你真相"，而在"诚实地处理自己的盲区"。
>
> 这把产品所有的限制（decoder 保守、输盘读不到、样本偏差）收编成一个主动选择的立场，而非一堆要道歉的缺陷。

## 第八类 · v3 愿景（从"96% 赢家离场"发现往前推）

> ⚠️ **时机纪律**：以下全部是 v3 的料，**不在 v2 lift 收尾前碰**。而且——v3 这些方向值不值得做，取决于 v2 的 lift 先回答"decoder 判断力到底成不成立"。大脑还没验准，别急着给它装更多传感器。先让 v2 告诉你这个大脑行不行，再决定 v3 装什么。

### 愿景 A · 从"看持仓"转向"看动作"（范式转向）
- **触发认知**：96% 赢家离场，证明最有价值的信息不在"持有"里，在"进出"里。v1/v2 是"快照式"（抓此刻持有什么），但聪明钱的四个动作各自都是强信号，目前一个都没读：
  - 建仓 = 开始相信 / 加仓 = 信念增强 / 赎回 = edge 已尽该走 / 割肉离场 = 认输看错
- **转向**：v3 范式从"持仓快照"→"行为流"。不问"他现在拿着什么"，问"他刚做了什么动作、意味着什么"。
- **反转盲区为信号源**：赢家"消失"让你看不到持仓，但"离场"这个动作本身是可捕捉的交易事件——把最大的盲区（赢家消失）变成最强的信号（"该走了"）。接 #25。
- **「离场盈亏」才是输赢的正确口径（v2 lift 收尾时定调，路 B）**：v2 lift 被迫用「结算结果」定义输赢（持有侧==winner），这要求样本"持到结算"，于是系统性排除提前离场的聪明钱、且大额输盘过采稀有滞留者——偏差是定义本身带来的，扫再多盘也消不掉。正确口径是**聪明钱自己的离场盈亏**（建仓价→离场价，这一注他到底赚没赚），每笔 round-trip 都是合法样本、无需持到结算，幸存者偏差从根上消失，且与跟单者真实行为（择机止盈离场、不持到结算）一致。
  - **为何归到这里**：算离场盈亏 = 必须重建每个聪明钱的完整进出轨迹 = 就是愿景 A 的"看动作"工程，二者同一件事。故 v3 做"行为流"时，lift 口径自然从「结算结果」升级到「离场盈亏」。
  - **v2 现状（务实退路·路 A）**：用「结算结果」定义 + 现有 B 路径出带 caveat 的 lift，只回答"decoder 判断力成不成立"；不强行在 v2 收尾期开"离场盈亏"这个大工程（破坏主线）。对应 #26 的"方向对≠能赚到钱"——结算口径连"赚没赚"都没真正度量，离场盈亏才是。
  - **🔺 v2 数据已将路 B 升级为「v3 首要任务」（2026-06-14）**：v2 lift 实测证明——结算输赢口径在
    edge-band 只挤出 3 个 GO、喂不饱，decoder 的「硬盘 edge」测不出。离场盈亏是唯一能喂饱 GO 子集的口径，
    不再是「可选优化」，是 **v3 第一仗**。详见顶部「v2 结算结论」+ `backtest/lift_v1.md`。

### 愿景 B · 从"解读单点"升级到"追踪轨迹"（产品形态升级）
- **触发认知**：聪明钱的 alpha 不在某一注，在完整决策轨迹（何时进、何价加、看到什么走）。
- **升级**：解决的问题从"这一注在赌什么、还能不能跟"→"追踪一个聪明钱的完整下注生命周期，在每个关键动作点提醒你"。从"我帮你读懂这一注"→"我帮你像影子一样跟住这个人"。
- **形态变化**：v3 可能不再是"输入钱包出张卡"，而是"订阅一个聪明钱，他一动你就知道"——从工具变哨兵。接 #13（无法实时）+ #25。

### 愿景 C · 终极问题不是"跟谁"，是"信谁"（核心引擎升级）
- **触发认知**：v2 全程真正在打磨的核心能力，不是"解读下注"，是"判断可信度"（红线、防泄漏、去偏差、辨真假催化剂）。预测市场里真正稀缺的不是信息，是"哪些信息/哪些人值得信"。
- **升级为"可信度引擎"**：
  - 不只"聪明钱买了什么"，而是"这个聪明钱在**这类盘**上历史上靠不靠谱"（有人擅长地缘、有人擅长选举，不该一视同仁）
  - 不只给新闻，而是"这条新闻是真催化剂还是会骗你的噪音"（decoder 已有雏形，如 Starmer 那条骗人的辞职压力新闻）
  - 不只算 lift，而是"什么条件下我的判断可信、什么条件下我自己也没把握"（置信度的下一代）
- **本质**：卖的从来不是"信号"，是"信号的可信度"。是定位句"卖清醒不卖确定"的终极工程化形态。

### v3 北极星（一句话）
> v1 解读单注 → v2 验证判断力 → **v3 追踪聪明钱的完整行为轨迹，并为每个动作、每个信号、每个人标注"可信度"——做一个"在噪音海洋里告诉你该信谁、信到什么程度"的可信度引擎。**
